Executive Summary

In this project, I looked to design and implement a wireless mouse unit with gyroscope based movement control. The aims of this project were to simulate cursor motion/click mechanisms simply by using hand gestures; by moving the unit freely in the air. The challenge was to give the user the ability to operate any computer unit as their hands are in midair; this would mean taking wires and surfaces out of the picture.
 
To make this possible, the STM32F103C8T6 [Blue Pill] was used as the main microcontroller as it will be programmed for the interference with 2 push buttons [Left, Right], Bluetooth Module HC-05 [For Wireless Communication], and an MPU-6050 [Gyroscope/Accelerometer]. In addition, a Buck Converter [DC-to-DC] would be used to step down the voltage of the unit; doing so to reduce the unit’s heat and extend the battery/lifetime. For the finishing touch, it will be necessary to provide the user dynamic feedback; so an OLED screen will be put on the wireless unit showing realtime sensor data points.
 
Now, let’s take a look at the process flow; as the Bluetooth Module will be interfaced with the Blue Pill, it will be able to establish UART communication [Receive and Transmit] with the preferred laptop/computer the bluetooth is paired to. When the unit is on, the MPU-6050 IMU sensor will give it’s dynamic positional values, as they will be used by the Blue Pill Microcontroller. These values will then accordingly give instructions to the laptop/computer to move the cursor dynamically in the desired direction; it will also send serial print data [Such as: Left, Right, Up, Down, Left Click, Right Click, Double Click] to the laptop/computer through bluetooth. The user will be provided the realtime X-Y coordinates dynamically on the OLED display on the wireless unit, along with a moving circle graphic which will correspond to the X-Y movement.
 
Putting the picture together, the user will be able to simulate mouse movement by performing an X-axis Pitch [Rotating unit Left or Right] , as well as a Y-axis Roll [Rotating unit Forwards or Backwards] or [Both Together]. (Figures shown to the right) This will coincide with the 2 push buttons [Left, Right] available to perform the operations of Left Click, Right Click, and Double Click.

Design Overview

With empasis on simplicity of use, the unit should be able to be used by relatively any user who has some insight of operating a computer and using a mouse. The aim is to give that user the ability to control the mouse away from the actual machine, whether they are standing, sitting, laying, etc; as the relative space in front of them would serve as the ‘mousepad’ for them to rotate the unit.
 
In the design of the unit: the gyroscope/accelerometer data values will simulate only the cursor movement (X-Y-axis), as it was troublesome to get it working in its Z-axis direction (for clicking mechanism). In attempts to make this possible, the data rate being captured for this axis would either zero out, and sometimes even fluctuate highly when moved. In this situation, it was not feasible to even program a threshold value the Z-axis would have to reach relative to its position to initiate a click function for the mouse. This is the point where 2 push buttons were incorporated to perform the [Left Click, Right Click, Double Click] functions.
 
This modification is what leaves the MPU-6050 in charge of capturing the data values of the tilt only in the X-Y-axis, as the HC-05 module would communicate these values to correspond to the movement of the cursor itself. [As the OLED would display these values for the user dynamically]

Retrospective

Overall, the key points were hit in the creation of the AGM: Accuracy, Speed of Execution, Consistency, Usability, and Safety. The prototype is fully capable of effectively giving the user control of a mouse, in terms of cursor movement and clicking as they are also provided dynamic feedback of relative position.
 
The key here was to ensure a fully wireless mouse unit, which could communicate via Bluetooth, and create input values for movement based on rotation of the unit in the air; all this while keeping the size of the unit small and safe from any dangerous charge. As the creation of the unit progressed, the real challenge was to give the user control without having to press anything [utilizing Z-axis click]; though with the constraints involved this functionality was implemented using push buttons.
 
In attempts to break the existing paradigm of a computer mouse, I believe there is a substantial potential for this unit to grow. More specifically, extending the customization ability to the user in terms of swipe/vibration gestures, minimizing the size of the unit, higher resolution cursor movement, as well as manufacturing a sleek shell this unit can reside in. The greatest challenge for enhancing this unit is to base the cursor movement purely off of 2-D positioning using accelerometer movement readings; though this is a mission for the next chapter.
 
In entirety, I am completely satisfied, but not at all complacent with the AGM; yet.
